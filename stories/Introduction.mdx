<div className='subheading'>Github Actions examples</div>

- Build and Deploy with Git Action:

<Source
  dark
  language='yml'
  code={`
  on:
  release:
    types: [published]

name: Build and Deploy

jobs:
build-and-deploy:
name: Build and Deploy
runs-on: ubuntu-latest

    steps:
      - name: Checkout GitHub Action
        uses: actions/checkout@master

      - name: Make envfile
        run: |
          cat > .env
          echo "VITE_VERSION=v1.0.0 >> .env
          echo "VITE_V_DATE=$(date +'%Y-%m-%dT%H:%M:%S')" >> .env
          echo "$(<.env)"

      - name: Setup Node
        uses: actions/setup-node@master
        with:
          node-version: 20.1

      - name: Install PNPM
        run: npm install -g pnpm

      - name: Install NPM Login
        run: npm install -g npm-cli-login

      - name: NPM Login
        run: NPM_USER=\${{ vars.NPM_USER }} NPM_PASS=\${{ secrets.NPM_PASS }} NPM_EMAIL=\${{ vars.NPM_EMAIL }} NPM_REGISTRY=\${{ vars.NPM_URL }} NPM_SCOPE=@j-meira npm-cli-login

      - name: Install depends
        run: |
          pnpm install --ignore-scripts

      - name: Build
        run: |
          pnpm build

      - name: Deploy step
        run: //any action to deploy

`}
/>

- Build with Docker in Git Action:

<Source
  dark
  language='yml'
  code={`
  on:
  release:
    types: [published]

name: Build with Docker

jobs:
build-and-deploy:
name: Build and Deploy
runs-on: ubuntu-latest

    steps:
      - name: Checkout GitHub Action
        uses: actions/checkout@master

      - name: Login Docker
        uses: docker/login-action@master
        with:
          username: \${{ secrets.DOCKERHUB_USER }}
          password: \${{ secrets.DOCKERHUB_TOKEN }}

      - name: Make envfile
        run: |
          cat > .env
          echo "VITE_VERSION=v1.0.0 >> .env
          echo "VITE_V_DATE=$(date +'%Y-%m-%dT%H:%M:%S')" >> .env
          echo "$(<.env)"

      - name: Build and Push Docker Image
        run: |
          docker build . -t \${{ vars.REPOSITORY_NAME }}:\${{ env.TAG_NAME }} --build-arg NPM_USER=\${{ vars.NPM_USER }} --build-arg NPM_EMAIL=\${{ vars.NPM_EMAIL }} --build-arg NPM_PASS=\${{ secrets.NPM_PASS }} --build-arg NPM_URL=\${{ vars.NPM_URL }}
          docker push \${{ vars.REPOSITORY_NAME }}:\${{ env.TAG_NAME }}

`}
/>

- DockerFile:

<Source
  dark
  language='bash'
  code={`
  # Stage 0, "build-stage"
FROM node:20.1 as build-stage
ARG NPM_EMAIL
ARG NPM_USER
ARG NPM_PASS
ARG NPM_URL

WORKDIR /app

COPY ./nginx.conf /nginx.conf

COPY package\*.json /app/

RUN npm install -g pnpm

RUN npm install -g npm-cli-login

RUN npm-cli-login -u \${NPM_USER} -p \${NPM_PASS} -e \${NPM_EMAIL} -r \${NPM_URL} -s "@j-meira"

RUN pnpm install --ignore-scripts

COPY ./ /app/

RUN pnpm build

# Stage 1, "deploy"

FROM nginx:1.22.1 as deploy-stage

COPY --from=build-stage /app/dist/ /usr/share/nginx/html

COPY --from=build-stage /nginx.conf /etc/nginx/conf.d/default.conf
`}
/>

<Source

  code={`# MUI-Theme v2 Migration Guide

## Breaking Changes

### 1. MUI Core v7 Upgrade

- MUI Grid v2 is now the default (old Grid props removed)
- \`md\`, \`lg\`, \`xl\` props replaced with \`size\` prop
- Breaking changes in Date Pickers (v8.19.0)

### 2. React 19 Compatibility

- Updated prop validation (remove invalid DOM props)
- Enhanced type safety requirements

### 3. Vite 7

- New build configuration
- Updated dependencies

## Migration Steps

### Step 1: Update Dependencies

\`\`\`bash

# Update mui-theme

pnpm update @j-meira/mui-theme@latest

# Update peer dependencies

pnpm add react@^19.0.0 react-dom@^19.0.0

# Update MUI packages (if using directly)

pnpm add @mui/material@^7.3.5 @mui/x-date-pickers@^8.19.0
\`\`\`

### Step 2: Fix MUI Grid Props

Replace old Grid props with new \`size\` prop:

\`\`\`typescript
// ❌ Before (v1)

<Grid xs={12} md={6} lg={4}>
  <Component />
</Grid>

// ✅ After (v2)

<Grid size={{ xs: 12, md: 6, lg: 4 }}>
  <Component />
</Grid>

// Or use single size for all breakpoints

<Grid size={12}>
  <Component />
</Grid>
\`\`\`

### Step 3: Update DatePicker Usage

\`\`\`typescript
// The DatePicker component API remains the same
// but internal MUI changes may affect custom implementations
import { DatePicker } from '@j-meira/mui-theme';

// ✅ Still works

<DatePicker label='Select Date' value={date} onChange={handleChange} />
\`\`\`

### Step 4: Remove Invalid Props

\`\`\`typescript
// ❌ Before (causes React 19 warnings)

<input action='/submit' searchChange={handler} />

// ✅ After

<form action='/submit'>
  <input onChange={handler} />
</form>
\`\`\`

### Step 5: Run Tests

\`\`\`bash

# Verify everything works

pnpm test

# Check for type errors

pnpm tsc --noEmit

# Build to ensure no runtime errors

pnpm build
\`\`\`

## Component-Specific Changes

### No Breaking Changes:

- ✅ Button (all variants)
- ✅ DataTable (all components)
- ✅ Input components (Basic, CheckBox, Currency, etc.)
- ✅ Hooks (useCookies, useToast, useDebounce, etc.)
- ✅ MultiProvider
- ✅ Dialog, PopUp, Header, SideBar, etc.

### Internal Updates Only:

- DatePicker: Updated to MUI v8 pickers (API unchanged)
- SearchGeneric/SearchRequest: Code optimizations (no API changes)

## Migration Checklist

- [ ] Remove invalid DOM props (action, searchChange, etc.)
- [ ] Run tests and fix any failures
- [ ] Build project successfully
- [ ] Test in development environment
- [ ] Deploy to staging for QA
- [ ] Update documentation
